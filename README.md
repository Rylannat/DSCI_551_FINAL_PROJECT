## Project Overview

This project implements a Flight Search Application backed by PostgreSQL to explore and demonstrate how B-tree indexing, partial indexes, and covering indexes affect query execution performance. The application allows users to search for flights by route, date, departure time, and delay filters, while also exposing the live PostgreSQL query execution plan (EXPLAIN ANALYZE output) directly in the UI, making internal database behavior visible alongside search results.

Six features are implemented, each analyzed using EXPLAIN ANALYZE before and after index creation to demonstrate the performance impact of indexing on real flight data.

## Step 4 - Run the Data Cleaning Script

```bash
cd data
python clean_csv.py
```

This script reads all `.csv` files in the current directory and for each one:
- Standardizes column names to match the database schema
- Converts HHMM integer time fields to `HH:MM:SS` format
- Builds valid `fl_date` values with year/month/day fallback
- Generates a synthetic price: `price = (distance × 0.10) + (duration × 0.50)`
- Creates `flight_name` combining carrier code and flight number
- Saves output as `cleaned_<filename>.csv` inside `/data/cleaned/`

The cleaned files are already pre-generated in `/data/cleaned/`. You may skip this step and use them directly.

## Step 11 - Reproducing the Before/After Index Experiments

Keep **pgAdmin Query Tool** and **Streamlit UI** open side by side. For each feature, follow these four steps in order.

**Step 1 - EXPLAIN ANALYZE before index (Sequential Scan):**  
In pgAdmin, run the `EXPLAIN (ANALYZE, BUFFERS)` query that appears *before* the `CREATE INDEX` line in the SQL file for that feature. You will see a **Sequential Scan** with higher execution time.  
At the same time, enter the filter values from the table below in the Streamlit UI and click **Search**. The Query Execution Plan panel will also show **Sequential Scan**.

**Step 2 - Create the index:**  
Run the `CREATE INDEX` line in pgAdmin. No UI action needed.

**Step 3 - EXPLAIN ANALYZE after index (Index Scan):**  
Run the same `EXPLAIN (ANALYZE, BUFFERS)` query again in pgAdmin. You will now see an **Index Scan** or **Index Only Scan** with dramatically lower execution time.  
Enter the same filter values in the Streamlit UI and click **Search** again. The Query Execution Plan panel will now show **Index Scan**.

**Step 4 - Drop the index:**  
Run the `DROP INDEX IF EXISTS` line in pgAdmin to reset state before the next feature.
